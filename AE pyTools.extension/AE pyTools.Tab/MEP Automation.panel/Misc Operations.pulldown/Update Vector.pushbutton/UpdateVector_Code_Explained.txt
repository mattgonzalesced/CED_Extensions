Update Vector - Code Explained
==============================

Purpose
-------
`Update Vector` refreshes the geometric portion of an equipment definition
without rebuilding profile metadata. Designers use it after moving selected
placed elements in the model. The command recalculates offsets and rotation,
updates Extensible Storage YAML, and rewrites Element_Linker payload text on
the selected elements.

Execution Flow
--------------
1. Selection + Payload Parse - The command only processes currently selected
elements. It reads each element's Element_Linker payload and parses:
Linked Element Definition ID, Set Definition ID, Host Name, parent fields,
and location/rotation metadata.
2. Parent Resolution - Parent context is resolved in this order:
- Parent ElementId in the active model.
- Host Name match in the active model.
- Host Name match in linked models (transforming linked points/rotation to
  host-model coordinates).
- Stored Parent_location + Parent Rotation (deg) from the payload.
3. Offset Math - X/Y offsets and rotation are recomputed relative to the
resolved parent coordinate system:
- `x_inches`, `y_inches` from local delta to parent.
- `rotation_deg` from `child_rotation - parent_rotation`.
4. Z Behavior - Z is not overwritten for plan-only shifts.
- If no meaningful vertical movement is detected, existing YAML `z_inches`
  is preserved.
- If the element moved vertically, `z_inches` is recomputed from the element's
  level-relative elevation (not parent-relative Z drift).
5. YAML + Metadata Write - The script updates the matching
`linked_element_definitions[*].offsets[0]` entry in active YAML via
`save_active_yaml_data`, then rewrites Element_Linker text on the selected
elements so model metadata stays synchronized.

Notable Details
---------------
- Parent lookup reports which path was used (Parent ElementId, Host Name in
  active model, Host Name in linked model, or payload parent fallback).
- Element_Linker payload updates are wrapped in a Revit transaction, so updates
  remain undo/redo-safe.
- If a selected element cannot resolve valid parent context or LED mapping, it
  is skipped and summarized in the completion report.

In short, Update Vector is the geometric maintenance tool: it keeps YAML and
Element_Linker metadata aligned for selected elements after field layout edits.

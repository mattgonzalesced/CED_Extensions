Update Profiles - Code Explained
================================

Purpose
-------
Sync model changes back into the active YAML profile by scanning selected
elements with Element_Linker metadata and merging hosted annotations plus
parameter values into each linked element definition (LED).

Execution Flow
--------------
1. **Load YAML** - Pull the active YAML from Extensible Storage and index LED
   entries by ID.
2. **Scan Selection** - Require at least one selected element, then collect
   selected FamilyInstances/Groups that contain Element_Linker data and parse
   the LED ID from the payload.
3. **Capture Changes** - Reuse Manage Profiles helpers to collect hosted tags,
   keynotes, and text notes plus key parameters from each element. Nearby
   proximity capture also scans within 5 ft; keynote and text-note proximity
   capture only uses selected/highlighted annotation elements.
4. **Merge** - Add missing annotations, update offsets/parameters when values
   change, and record conflicts when multiple instances disagree.
5. **Save** - Write the updated YAML back to Extensible Storage so downstream
   placement tools use the newest profile data.

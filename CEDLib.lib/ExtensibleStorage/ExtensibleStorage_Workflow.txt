Extensible Storage Active YAML
------------------------------
The `ExtensibleStorage` module stores the active YAML payload and lightweight
metadata inside the RVT. It no longer keeps a change history; import/export
flows handle long-term storage.

Purpose
- Keep the active YAML text available to in-model tools.
- Store per-user settings (e.g., feature toggles).
- Store editor locks to prevent conflicting edits.

How it works
1. Tools load the active YAML via `ExtensibleStorage.get_active_yaml()` or the
   `ExtensibleStorage.yaml_store` helpers.
2. Tools update the active YAML via `update_active_yaml()` or
   `update_active_text_only()`.
3. Editor locks are stored in `meta["editor_lock"]` and released on sync.

Storage & sync details
- Data is stored in Extensible Storage on `DataStorage` using the v3 schema
  (fixed GUID) or earlier versions if already present.
- Chunked fields are used when metadata exceeds the Extensible Storage string
  limit.
